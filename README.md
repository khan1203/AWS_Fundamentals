# AWS Fundamentals

A structured, hands-on knowledge base documenting core AWS networking and infrastructure concepts, built while learning cloud fundamentals through practical implementation.

## 📖 About

This repository serves as a personal learning log and reference guide for AWS core services, with a focus on **networking fundamentals** — VPC design, subnetting strategy, and secure EC2 connectivity patterns. Each topic is documented as Markdown notes combining conceptual explanations with real-world troubleshooting scenarios encountered during hands-on setup.

The goal isn't just to follow steps, but to build accurate mental models of how AWS networking actually works under the hood.

## 🗂️ What's Covered

- **VPC Architecture** — Designing public/private subnet layouts, CIDR planning, and route tables
- **Internet & NAT Gateways** — Outbound internet access patterns for public vs. private resources
- **Security Groups** — Inbound/outbound rule configuration and common misconfigurations
- **EC2 Connectivity** — SSH access patterns, bastion hosts, and agent forwarding
- **Route Tables** — Local routing, internet routes, and common CIDR/routing mismatches
- **Troubleshooting Notes** — Real debugging scenarios (SSH key issues, permission errors, connectivity failures)

## 🎯 Purpose

This repo is intended for:
- Personal reference while learning AWS infrastructure
- Anyone else building foundational AWS networking knowledge
- Documenting common pitfalls and how to diagnose them

## 📁 Structure

Guides and notes are organized as `.md` files by topic, each covering theory alongside practical setup steps.
